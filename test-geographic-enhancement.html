地理信息增强功能测试

功能概述

地理信息增强服务专门用于为从火山引擎获取的事件补充缺失的经纬度数据，确保时间线生成功能的完整性。

🌍 智能地点识别

从事件地点字段和描述文本中智能提取地理位置信息

📍 坐标数据库

内置全球主要城市和国家的经纬度坐标数据

🔄 批量处理

支持批量为缺少坐标的事件补充地理信息

📊 统计监控

提供详细的处理统计和成功率监控

核心特性

1. 自动检测缺失坐标

系统会自动检测哪些事件缺少经纬度信息：

// 检测逻辑 public boolean needsGeographicEnhancement(Event event) { return event.getLatitude() == null || event.getLongitude() == null; }

2. 智能地点提取

从事件描述中提取地理位置信息：

// 正则表达式匹配 Pattern locationPattern = Pattern.compile( "(?:在|于|位于|发生在)\\s*([^，,。.\\s]{2,20}?)(?:[，,。.\\s]|$)" );

3. 坐标数据映射

内置主要城市和国家的坐标数据：

北京: [39.9042, 116.4074]
上海: [31.2304, 121.4737]
华盛顿: [38.9072, -77.0369]
伦敦: [51.5074, -0.1278]
东京: [35.6762, 139.6503]
...

4. 集成到时间线生成

在时间线生成过程中自动调用地理信息增强：

// 在EnhancedTimelineGenerationServiceImpl中 if (eventGeographicEnhancementService != null) { events = eventGeographicEnhancementService .enhanceEventDataListGeographicInfo(events); }

API接口测试

1. 获取增强统计

GET /api/geographic-enhancement/statistics

curl -X GET "http://localhost:8080/api/geographic-enhancement/statistics"

2. 为单个事件增强地理信息

POST /api/geographic-enhancement/enhance-event/{eventId}

curl -X POST "http://localhost:8080/api/geographic-enhancement/enhance-event/1"

3. 批量增强缺失的地理信息

POST /api/geographic-enhancement/enhance-missing?limit=100

curl -X POST "http://localhost:8080/api/geographic-enhancement/enhance-missing?limit=100"

4. 根据地点名称获取坐标

GET /api/geographic-enhancement/coordinates?locationName=北京

curl -X GET "http://localhost:8080/api/geographic-enhancement/coordinates?locationName=北京"

5. 从文本中提取地点信息

POST /api/geographic-enhancement/extract-location

curl -X POST "http://localhost:8080/api/geographic-enhancement/extract-location" \ -H "Content-Type: application/json" \ -d '{"text": "事件发生在北京市中心地区"}'

测试步骤

  1. 启动后端服务：确保Spring Boot应用正常运行
  2. 检查服务状态：访问统计接口确认服务可用
  3. 测试单个事件增强：选择一个缺少坐标的事件进行测试
  4. 测试批量增强：运行批量增强接口
  5. 验证时间线生成：创建时间线验证地理信息是否正确补充
  6. 检查数据库：确认事件表中的longitude和latitude字段已更新

验证要点

  • ✓ 缺少经纬度的事件能够自动补充坐标
  • ✓ 地点名称能够正确映射到坐标
  • ✓ 从事件描述中能够提取地点信息
  • ✓ 批量处理功能正常工作
  • ✓ 统计信息准确反映处理结果
  • ✓ 时间线生成包含完整的地理信息

数据库验证

检查事件表中的地理信息字段：

-- 查询缺少地理信息的事件 SELECT id, event_location, latitude, longitude, geographic_status FROM event WHERE latitude IS NULL OR longitude IS NULL; -- 查询已增强地理信息的事件 SELECT id, event_location, latitude, longitude, geographic_status, geographic_updated_at FROM event WHERE geographic_status = 1 ORDER BY geographic_updated_at DESC LIMIT 10;

性能监控

监控地理信息增强的性能指标：

  • 处理速度：每秒处理的事件数量
  • 成功率：成功获取坐标的事件比例
  • 缓存命中率：坐标查询的缓存效率
  • 错误率：处理失败的事件比例

故障排除

常见问题

  • 坐标未找到：检查地点名称是否在坐标数据库中
  • 地点提取失败：检查事件描述的格式是否符合正则表达式
  • 批量处理缓慢：考虑调整批量处理的数量限制
  • 数据库更新失败：检查数据库连接和事务配置
