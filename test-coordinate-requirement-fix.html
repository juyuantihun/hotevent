火山引擎API经纬度要求修复测试

问题描述

根据日志分析，发现火山引擎API调用时没有要求返回经纬度数据，导致生成的事件缺少地理坐标信息。

原始问题：没有要求火山引擎返回经纬度，应该加上要求返回经纬度

修复方案

在API调用的提示词中明确要求返回经纬度坐标，并更新JSON格式规范。

修复前的JSON格式

{ "events": [ { "title": "事件标题", "description": "详细描述", "eventTime": "2024-01-01T12:00:00", "location": "具体地点", "subject": "事件主体", "object": "事件客体", "eventType": "事件类型", "keywords": ["关键词1", "关键词2"], "sources": ["来源1", "来源2"], "credibilityScore": 0.95 } ] }

修复后的JSON格式

{ "events": [ { "id": "event_001", "title": "事件标题", "description": "事件详细描述", "eventTime": "2024-01-15T10:30:00Z", "location": "北京市", "latitude": 39.9042, "longitude": 116.4074, "subject": "事件主体", "object": "事件客体", "eventType": "政治", "keywords": ["关键词1", "关键词2"], "sources": ["来源1", "来源2"], "credibilityScore": 0.85 } ] }

修复详情

1. 修改DeepSeekServiceImpl.java

在 buildEventFetchPrompt 方法中更新了输出格式要求：

  • 添加了 latitude 和 longitude 字段要求
  • 明确要求经纬度必须是数值格式
  • 强调每个事件都必须包含坐标信息

2. 修改PromptTemplateServiceImpl.java

在 getEventFetchResponseFormat 方法中更新了响应格式：

  • 添加了详细的字段说明
  • 提供了完整的JSON示例
  • 强调了经纬度的重要性和格式要求

3. 新增的重要要求

=== 重要要求 === 1. **必须提供准确的经纬度坐标**：每个事件都必须包含latitude和longitude字段 2. 经纬度必须是数值格式（如：39.9042），不能是字符串 3. 如果无法确定精确坐标，请提供该地区的中心坐标或主要城市坐标 4. 返回的JSON必须格式正确，可以直接解析 5. 请严格按照上述格式返回，不要添加任何额外的文字说明

修复的服务文件

  • ✓ DeepSeekServiceImpl.java - 主要的DeepSeek服务实现
  • ✓ PromptTemplateServiceImpl.java - 提示词模板服务

这两个文件都被修改以确保API调用时明确要求返回经纬度坐标。

测试步骤

  1. 重启应用：确保修改的代码生效
  2. 创建时间线：通过前端或API创建新的时间线
  3. 检查API调用：查看日志确认提示词包含经纬度要求
  4. 验证响应数据：确认返回的事件包含latitude和longitude字段
  5. 检查数据库：验证事件表中的经纬度字段是否有值

验证要点

  • ✓ API提示词包含经纬度要求
  • ✓ 返回的JSON包含latitude和longitude字段
  • ✓ 经纬度值是数值格式，不是字符串
  • ✓ 每个事件都有坐标信息
  • ✓ 数据库中的事件记录包含经纬度数据
  • ✓ 时间线生成功能正常工作

数据库验证

检查事件表中的经纬度字段：

-- 查询最近创建的事件及其坐标信息 SELECT id, event_location, latitude, longitude, source_type, created_at FROM event WHERE created_at >= DATE_SUB(NOW(), INTERVAL 1 HOUR) ORDER BY created_at DESC LIMIT 10; -- 统计有坐标的事件数量 SELECT COUNT(*) as total_events, COUNT(CASE WHEN latitude IS NOT NULL AND longitude IS NOT NULL THEN 1 END) as events_with_coordinates, ROUND(COUNT(CASE WHEN latitude IS NOT NULL AND longitude IS NOT NULL THEN 1 END) * 100.0 / COUNT(*), 2) as coordinate_coverage_percent FROM event WHERE source_type = 1; -- DeepSeek获取的事件

故障排除

如果仍然缺少坐标

  • 检查API响应是否包含latitude和longitude字段
  • 验证提示词是否正确发送给火山引擎
  • 查看API响应解析是否正确处理坐标字段
  • 确认地理信息增强服务是否正常工作

如果坐标格式错误

  • 检查API返回的坐标是否为字符串格式
  • 验证JSON解析是否正确转换数据类型
  • 查看数据库字段类型是否支持decimal格式
