记住账号功能修改测试

修改内容

将登录页面的"记住密码"文本修改为"记住账号"，更准确地反映功能实际作用。

修改前

<el-checkbox v-model="loginData.rememberMe"> 记住密码 </el-checkbox>

修改后

<el-checkbox v-model="loginData.rememberMe"> 记住账号 </el-checkbox>

功能说明

该功能的实际作用是：

  • ✓ 记住用户输入的用户名（账号）
  • ✓ 下次访问登录页面时自动填充用户名
  • ✓ 自动勾选"记住账号"复选框
  • ✗ 不会记住密码（出于安全考虑）

注意：系统不会记住密码，每次登录都需要重新输入密码，这是为了保证账户安全。

测试步骤

  1. 清除浏览器的本地存储数据
  2. 访问登录页面
  3. 输入用户名和密码
  4. 勾选"记住账号"复选框
  5. 点击登录按钮
  6. 登录成功后，退出登录
  7. 重新访问登录页面
  8. 验证用户名是否自动填充
  9. 验证"记住账号"复选框是否自动勾选
  10. 验证密码输入框是否为空

验证要点

  • ✓ 登录页面显示"记住账号"而不是"记住密码"
  • ✓ 勾选后能够记住用户名
  • ✓ 不勾选时不会记住用户名
  • ✓ 密码永远不会被记住
  • ✓ 页面刷新后记住的用户名仍然有效

技术实现

该功能的技术实现原理：

1. 前端存储

// 存储记住的用户名 localStorage.setItem('rememberedUsername', username) // 读取记住的用户名 const rememberedUsername = localStorage.getItem('rememberedUsername')

2. 自动填充逻辑

// 在页面加载时检查是否有记住的用户名 onMounted(async () => { const rememberedUsername = localStorage.getItem('rememberedUsername') if (rememberedUsername) { loginData.username = rememberedUsername loginData.rememberMe = true } })

3. 登录时处理

// 根据用户选择决定是否记住用户名 if (loginData.rememberMe) { setRememberedUsername(loginData.username) } else { setRememberedUsername('') }

安全考虑

为什么只记住账号而不记住密码：

  • 安全性：密码是敏感信息，不应该存储在浏览器中
  • 合规性：符合安全最佳实践
  • 用户体验：减少重复输入用户名的麻烦
  • 风险控制：即使设备被他人使用，也需要输入密码才能登录
